# Pawboo v2

반려묘와 반려견을 키우는 보호자(집사)들이 자신의 반려동물을 자랑하고, 일일 미션을 통해 특별한 순간을 기록하며, 커뮤니티 내에서 소통하는 **반려동물 특화 SNS 플랫폼**입니다.

## 핵심 가치

- **미션 기반 게이미피케이션**으로 지속적인 참여 유도
- **반려동물 캐릭터 중심**의 커뮤니티 문화 형성
- **사진 기록**을 통한 반려동물 라이프스타일 아카이빙

---

## 핵심 기능

| 기능 영역  | 기능명               | 설명                           |
|--------|-------------------|------------------------------|
| 계정 관리  | 회원가입 / 로그인        | 소셜 로그인 (카카오), 비로그인 서비스 열람 가능 |
| 펫 캐릭터  | 펫 프로필 생성          | 반려동물 이름, 사진, 종류 등록           |
| 미션 시스템 | 일일 미션 제시 및 사진 업로드 | 매일 새로운 컨셉 미션 제공, 사진 업로드로 참여  |
| 피드     | 홈 피드 / 미션 피드      | 3열 그리드 형태, 미션 및 일반 업로드 통합    |
| 캘린더    | 펫별 날짜 기록 캘린더      | 펫 단위로 날짜별 업로드 사진 확인          |
| 검색     | 펫 이름 검색           | 펫 이름 기반 검색 및 프로필 탐색          |

## 화면 구조 (IA)

하단 탭 네비게이션 기반으로 4개 주요 영역으로 구성됩니다.

| Depth 1 (탭) | Depth 2 (화면) | Depth 3 (세부 기능)                          |
|-------------|--------------|------------------------------------------|
| **온보딩**     | 로그인          | 카카오 소셜 로그인, 비로그인 상태로도 서비스 열람 가능          |
|             | 로그인 유도       | 업로드 등 로그인 필요 기능 접근 시 로그인 안내 (방식 미정)      |
| **홈 (피드)**  | 통합 피드        | 미션 업로드 + 일반 업로드 통합, 3열 그리드 (사진만 표시), 최신순 |
|             | 피드 상세        | 사진, 업로드한 펫 프로필 사진 + 이름, 업로드 날짜, 좋아요      |
|             | 업로드 버튼       | 화면 하단 원형 고정 버튼, 대표 펫으로 업로드               |
| **미션**      | 오늘의 미션       | 일일 미션 확인, 가이드 예시 이미지, 대표 펫 미션 완료 여부 표시   |
|             | 미션 참여 피드     | 오늘 미션을 업로드한 유저들의 3열 그리드 피드 (사진만 표시)      |
|             | 업로드 버튼       | 화면 하단 원형 고정 버튼, 대표 펫으로 미션 업로드            |
| **프로필**     | 내 프로필        | 내 펫 목록 (리스트형), 펫 추가 버튼, 펫 미등록 시 생성 유도 버튼 |
|             | 내 펫 캘린더      | 펫 선택 시 해당 펫의 월별 캘린더, 날짜 탭으로 게시물 모달 표시    |
|             | 게시물 모달       | 사진, 업로드 날짜, 삭제 버튼 (본인 펫)                 |
|             | 타인 펫 캘린더     | 검색에서 타인 펫 프로필 진입 시 동일한 캘린더 UI, 삭제 버튼 미노출 |
|             | 펫 프로필 관리     | 펫 이름 수정, 대표 사진 변경, 대표 펫 설정, 펫 추가/삭제      |
|             | 설정           | 알림, 계정 관리, 개인정보 처리방침, 로그아웃               |
| **검색**      | 펫 이름 검색      | 펫 이름 검색                                  |
|             | 검색 결과        | 검색된 펫 목록 (펫 사진 + 이름), 탭 시 타인 펫 캘린더로 이동   |

## 상세 기능 명세

### 회원/계정 관리 `AUTH-001 ~ AUTH-010`

| 항목       | 내용                                              |
|----------|-------------------------------------------------|
| 비로그인 접근  | 홈 피드, 미션 탭, 검색 탭 열람 가능                          |
| 로그인 유도   | 업로드 등 로그인 필요 기능 접근 시 로그인 안내 (방식 미정)             |
| 회원가입 방법  | 카카오 OAuth2.0                                    |
| 유저-펫 연결  | 유저 계정에 펫을 연결하여 관리, 게시물 및 미션은 펫 단위로 귀속           |
| 로그인 후 이동 | 로그인 완료 시 홈 화면으로 이동, 펫 미등록 시 프로필 생성 유도 버튼 노출     |
| 로그인 유지   | JWT Access Token (1시간) + Refresh Token (30일) 방식 |
| 회원 탈퇴    | 탈퇴 시 user 행 삭제, 보유한 펫/게시물/좋아요 전부 삭제             |

### 펫 캐릭터 관리 `PET-001 ~ PET-015`

| 항목       | 내용                                                 |
|----------|----------------------------------------------------|
| 펫 등록 수   | 계정당 최대 5마리 등록 가능                                   |
| 등록 필수 정보 | 펫 이름(2~15자), 종류(고양이/강아지), 대표 사진 1장                 |
| 대표 사진    | JPG/PNG/WEBP, 최대 10MB, 자동 리사이즈 (400×400px), CDN 저장 |
| 대표 펫 설정  | 유저당 대표 펫 1마리 설정, 업로드 시 대표 펫으로 귀속                   |
| 펫 프로필 표시 | 펫 사진, 이름만 표시                                       |
| 펫 삭제     | 펫 삭제 시 해당 펫의 게시물 기록도 함께 삭제 (복구 불가)                 |

### 일일 미션 시스템 `MISSION-001 ~ MISSION-020`

| 항목       | 내용                                                     |
|----------|--------------------------------------------------------|
| 미션 등록 주체 | DB 직접 삽입 (관리자 API는 추후 추가 예정)                           |
| 미션 활성화   | 매일 00:00 KST 자동 활성화 (날짜별 1개 보장, `scheduled_at` UNIQUE) |
| 참여 제한    | 펫 1마리당 하루 1회만 제출 가능, 수정 불가 (삭제 후 재제출 가능)               |
| 업로드 스펙   | 이미지: JPG/PNG/WEBP, 최대 10MB, 장당, 최대 5장, 영상 업로드 미지원(v2)  |
| 이미지 처리   | 서버에서 리사이즈 (최대 1080px, inside fit), WebP 변환, CDN 저장     |
| 완료 여부 표시 | 미션 탭에서 대표 펫의 오늘 미션 완료 여부 표시                            |
| 미션 삭제    | 캘린더 게시물 모달에서 삭제, 삭제 후 재제출 가능                           |

### 피드 및 게시물 `FEED-001 ~ FEED-015`

| 항목        | 내용                                       |
|-----------|------------------------------------------|
| 피드 유형     | 홈 통합 피드 (미션 + 일반 업로드), 미션 탭 당일 미션 참여 피드  |
| 그리드 표시    | 3열 그리드, 사진만 표시 (미션/일반 구분 없음), 최신순        |
| 피드 상세 표시  | 사진, 업로드한 펫 프로필 사진, 펫 이름, 업로드 날짜, 좋아요     |
| 페이지네이션    | 무한 스크롤, 페이지당 20개 로드                      |
| 업로드 플로우   | 이미지 선택/촬영 최대 5장, 대표 펫으로 귀속 (추가 입력 항목 미정) |
| 업로드 버튼    | 홈, 미션 탭 하단 원형 고정 버튼                      |
| 좋아요       | 피드 상세에서 하트 아이콘 탭, 중복 불가, 취소 가능           |
| 게시물 삭제    | 캘린더에서 사진 탭 시 모달 표시, 모달 내 삭제 버튼 (본인 펫만)   |
| 게시물 신고    | 부적절한 사진/스팸/저작권 위반 사유 선택 신고               |
| 게시물 공개 범위 | v2: 전체 공개만 지원 (추후 선택 공개 옵션 추가 예정)        |

### 캘린더 `CALENDAR-001 ~ CALENDAR-010`

| 항목     | 내용                                       |
|--------|------------------------------------------|
| 캘린더 단위 | 펫별 캘린더 (유저 공통 캘린더 없음)                    |
| 표시 정보  | 월별 캘린더, 게시물이 있는 날짜에 대표 사진 썸네일 표시         |
| 날짜 탭   | 날짜 탭 시 해당 날짜 업로드 사진 모달로 표시               |
| 게시물 모달 | 사진, 업로드 날짜 표시, 본인 펫이면 삭제 버튼 노출           |
| 타인 열람  | 검색에서 타인 펫 프로필 진입 시 동일한 캘린더 UI, 삭제 버튼 미노출 |

### 검색 `SEARCH-001 ~ SEARCH-005`

| 항목    | 내용                                    |
|-------|---------------------------------------|
| 검색 대상 | 펫 이름만 검색 가능                           |
| 검색 결과 | 펫 이름 일치 목록 (펫 사진 + 이름), 탭 시 펫 캘린더로 이동 |

## 개발 순서

### Phase 1 — 인증 / 계정

1. 카카오 OAuth2.0 로그인 / 회원가입
2. JWT Access Token + Refresh Token 발급 및 갱신
3. 비로그인 열람 처리 및 로그인 유도 (방식 미정)
4. 로그아웃
5. 회원 탈퇴

### Phase 2 — 펫 캐릭터

1. 최초 펫 등록 (로그인 후 유도 플로우)
2. 펫 프로필 조회 / 수정 / 삭제
3. 펫 추가 등록 (최대 5마리)
4. 대표 펫 설정 및 전환

### Phase 3 — 미션 시스템

1. 관리자 미션 등록 (제목, 설명, 예시 이미지, 예약 발행)
2. 오늘의 미션 조회 및 대표 펫 완료 여부 확인
3. 미션 사진 제출 (이미지 리사이즈, CDN 저장)
4. 제출 취소 (재제출 가능)

### Phase 4 — 피드

1. 홈 통합 피드 (3열 그리드, 사진만, 최신순, 무한 스크롤)
2. 피드 상세 (펫 사진 + 이름, 업로드 날짜, 좋아요)
3. 미션 탭 당일 미션 참여 피드
4. 좋아요 / 취소
5. 게시물 신고

### Phase 5 — 캘린더

1. 펫별 월별 캘린더 (날짜 썸네일)
2. 날짜 탭 시 게시물 모달
3. 모달 내 게시물 삭제 (본인 펫만)
4. 타인 펫 캘린더 열람 (삭제 버튼 미노출)

### Phase 6 — 검색

1. 펫 이름 검색 및 결과 목록
2. 검색 결과에서 펫 캘린더로 이동

## 데이터 모델

### user

| 컬럼         | 타입             | 설명            |
|------------|----------------|---------------|
| id         | uuid PK        | 유저 고유 ID      |
| kakao_id   | varchar UNIQUE | 카카오 OAuth 식별자 |
| created_at | timestamptz    | 가입일시          |

> 탈퇴 시 user 행 삭제, 연결된 pet, post, post_like CASCADE 삭제

### pet

| 컬럼                | 타입             | 설명                |
|-------------------|----------------|-------------------|
| id                | uuid PK        | 펫 고유 ID           |
| user_id           | uuid FK → user | 소유 유저             |
| name              | varchar(15)    | 펫 이름              |
| image_url         | varchar        | 대표 사진 CDN URL     |
| is_representative | boolean        | 대표 펫 여부 (유저당 1마리) |
| created_at        | timestamptz    | 등록일시              |

> UNIQUE INDEX (user_id) WHERE is_representative = true — 유저당 대표 펫 1마리 보장
> pet 삭제 시 해당 펫의 post CASCADE 삭제

### mission

| 컬럼           | 타입          | 설명                    |
|--------------|-------------|-----------------------|
| id           | uuid PK     | 미션 고유 ID              |
| title        | varchar     | 미션 제목                 |
| description  | text        | 미션 설명                 |
| scheduled_at | date UNIQUE | 미션 활성화 날짜 (날짜당 1개 보장) |
| created_at   | timestamptz | 등록일시                  |

### post

| 컬럼         | 타입                        | 설명                                         |
|------------|---------------------------|--------------------------------------------|
| id         | uuid PK                   | 게시물 고유 ID                                  |
| pet_id     | uuid FK → pet             | 업로드한 펫                                     |
| type       | enum('general','mission') | 일반 업로드 / 미션 업로드 구분                         |
| mission_id | uuid FK → mission NULL    | 미션 업로드일 때만 연결, 일반이면 NULL                   |
| image_urls | varchar[]                 | 이미지 CDN URL 배열 (최대 5장, image_urls[0]이 썸네일) |
| created_at | timestamptz               | 업로드일시                                      |

> UNIQUE INDEX (pet_id, mission_id) WHERE mission_id IS NOT NULL — 펫당 미션 중복 제출 방지
> CHECK array_length(image_urls, 1) BETWEEN 1 AND 5 — 이미지 1~5장 제한

### post_like

| 컬럼         | 타입                      | 설명                 |
|------------|-------------------------|--------------------|
| id         | uuid PK                 | 좋아요 고유 ID          |
| post_id    | uuid FK → post          | 대상 게시물             |
| user_id    | uuid FK → user NOT NULL | 좋아요 누른 유저 (로그인 필수) |
| created_at | timestamptz             | 좋아요 일시             |

> UNIQUE (post_id, user_id) — 중복 좋아요 방지

### report

| 컬럼          | 타입                                       | 설명        |
|-------------|------------------------------------------|-----------|
| id          | uuid PK                                  | 신고 고유 ID  |
| post_id     | uuid FK → post                           | 신고 대상 게시물 |
| reporter_id | uuid FK → user                           | 신고한 유저    |
| reason      | enum('inappropriate','spam','copyright') | 신고 사유     |
| created_at  | timestamptz                              | 신고일시      |
